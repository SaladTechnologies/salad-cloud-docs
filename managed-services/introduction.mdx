## API Reference

### Send a POST request with job parameters

To interact with the API, you first need to send a POST request to the endpoint url mentioned above. Salad API Key should be sent as a header: "Salad-Api-Key": "Your Salad Key". As a result you will recieve a job id. Follow the pattern below for the request body:

```json
{
  "input": {
    "url": "https://example.com/path/to/file.mp3",
    "language_code": "en",
    "word_level_timestamps": true,
    "diarization": true,
    "srt": true
  },
  "metadata": {
    "my-job-id": 1234
  }
}

```

## Parameters

### input
An object that configures the job.
- **url**: Publicly accessible URL of the media file to transcribe.
- **language_code**: Language code of the media. Default is "en". Available options include:
  - "en" (English)
  - "fr" (French)
  - "de" (German)
  - "es" (Spanish)
  - "it" (Italian)
  - "ja" (Japanese)
  - "zh" (Chinese)
  - "nl" (Dutch)
  - "uk" (Ukrainian)
  - "pt" (Portuguese)
  - "ar" (Arabic)
  - "cs" (Czech)
  - "ru" (Russian)
  - "pl" (Polish)
  - "hu" (Hungarian)
  - "fi" (Finnish)
  - "fa" (Persian)
  - "el" (Greek)
  - "tr" (Turkish)
  - "da" (Danish)
  - "he" (Hebrew)
  - "vi" (Vietnamese)
  - "ko" (Korean)
  - "ur" (Urdu)
  - "te" (Telugu)
  - "hi" (Hindi)
  - "ca" (Catalan)
  - "ml" (Malayalam)
- **word_level_timestamps**: Set to true to get word-level timestamps. Default is false, which gets sentence-level timestamps.
- **diarization**: Set to true for speaker identification. Default is false.
- **srt**: Set to true to generate .srt captions. Default is false.

### metadata
Optional object to identify your unique transcription job.

## Example cURL Send for Transcription

```bash
curl -X POST https://api.salad.com/api/public/organizations/{my-organization}/inference-endpoints/transcribe/jobs \
   -H "Salad-Api-Key: "your salad api key" \
   -d '{
      "input": {
         "url": "https://example.com/path/to/file.mp3",
         "language_code": "en",
         "word_level_timestamps": true,
         "diarization": true,
         "srt": true
      }
   }'
```



## Example cURL Get Transcript
Once you have the job ID, you need to send a GET request with that job ID appended to the URL to retrieve the output. The pattern for the GET request is as follows:


```bash
curl -X GET "https://api.salad.com/api/public/organizations/{your-organization}/inference-endpoints/transcribe/jobs/{job_id}" \
   -H "Salad-Api-Key: {your-api-key}"
```

- **Example: Using the Transcription API via Postman**
  - Open postman and create a new HTTP request
  - Set the HTTP method to POST
  - Enter the API endpoint URL
  - Add API Key Authorization
    - Key: Salad-Api-Key
    - Value: provide your Salad API Key
    - Add to: header
  - Add the Request Body
    - select the raw option, and choose JSON from the dropdown menu
    - Add the URL of your audio file and parameters in the request body as shown in the example.
      - the file URL must be a direct download link
  - Send the Request
    - Click the Send button. The API will respond with the job id
  - Send another GET request with that job id at the end of the URL
    - Click the Send button. The API will respond with the transcription result in JSON format.
